================================================================================
PUSH PATTERN IN ONE 60-SECOND CYCLE
================================================================================

Total pushes per cycle: 5-6 separate HTTP POST requests

Typical timing within a cycle:
  t+0s:  amd-gpu-exporter pushes
  t+3s:  node-exporter pushes
  t+7s:  oci_lens_drhpc_metrics pushes
  t+8s:  oci_lens_node_metrics pushes
  t+17s: oci_lens_pod_metrics pushes

Healthcheck: Only runs when Kubernetes Job executes (not every cycle)

================================================================================
STRESS TESTING RECOMMENDATIONS
================================================================================

To accurately simulate real load:

1. REALISTIC TEST: Push all 5 payloads sequentially with 1-5 second delays
   between them, repeat every 60 seconds

2. HEAVY LOAD TEST: Push all 5 payloads concurrently, repeat at faster
   intervals (e.g., every 10-30 seconds)

3. SPIKE TEST: Push all 5 payloads from multiple simulated nodes
   simultaneously to test concurrent load

4. SUSTAINED LOAD: Run test #1 or #2 for extended periods (hours/days)

Total payload size per cycle: ~1.4 MB (sum of all 5 regular pushes)
